empty on day one — feel free to delete it once you've written something real.

## Why a personal site

A personal site is a place to think out loud: to keep a record of what I'm
working on, share the projects and papers I'm proud of, and occasionally write
about ideas that don't fit anywhere else.

## Writing a new post

Every post is a Markdown file in the `_posts/` folder, named
`YYYY-MM-DD-title.md`, with a little front matter at the top:

```yaml
---
title: "Your post title"
date: 2026-07-15
tags: [research, notes]
excerpt: "A one-line summary shown in the blog list."
---
```

Everything below the front matter is regular Markdown. You get headings,

> block quotes for the good lines,

`inline code`, fenced code blocks with syntax highlighting, lists, links — the
usual. Here's a code block to prove the styling works:

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"

print(greet("world"))
```

That's it. Push to `main` and GitHub Pages rebuilds the site automatically.
Happy writing.
